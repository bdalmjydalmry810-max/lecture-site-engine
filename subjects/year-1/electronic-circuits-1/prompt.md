### مراجعة سريعة (سؤال / جواب) — ضمن qa_cards / qa_inline / theory (≥12، easy)

— بدون تعليل طويل. مناسب للحفظ السريع قبل الامتحان.
مثال: "ما الفرق بين `مقاومة` و `ممانعة`؟"

### تحدي مركّب (سيناريو + أسئلة متعددة) — ضمن mcq / theory (≥3، hard)
### السيناريو {N}: {عنوان}
> [وصف الحالة: دارة، جدول، أو مسألة رياضية كاملة — 5–10 أسطر]
ثم 3–5 أسئلة MCQ **كلها مبنية على نفس السيناريو** (لا تكرر نفس الفكرة):
### السؤال {N}.1 (hard) … أ) ب) ج) د) + تعليل
كل سيناريو يختبر: فهم + تطبيق + قرار (أي خطوة تالية؟ أي ناتج؟ أين الخطأ؟)
المصادر: `numerical_case`, `circuit_state`, `waveform_analysis`
مثال: "دارة تحتوي بطارية ومقاومات موصولة على التوالي والتوازي — احسب التيار في كل فرع."

### تطبيق عملي (أكمل / احسب / حلّل) — ضمن exercise / analysis_exercise (≥5، medium)
كل تمرين: **المعطيات** + **المطلوب** (مرقّم) + **نموذج الحل الكامل**
الطالب يجيب بقيمة/جدول/مخطط — الإجابة قابلة للمقارنة حرفياً أو رقمياً.
الأنواع: `fill_gaps`, `table_fill`, `numerical_solve`, `diagram_completion`

### تتبع خطوة بخطوة (جدول ناقص) — ضمن trace_exercise / mcq (≥3، medium)
**المدخل:** {حالة ابتدائية}
**أكمل الجدول:**
| الخطوة | العملية | الحالة |
| 1 | ... | ؟ |
**نموذج الحل:** جدول مكتمل + **النتيجة النهائية**

الأهداف: `variable_state`, `signal_waveform`, `circuit_simulation_step`

### مقارنة وتحليل (أيهما أفضل؟ لماذا؟) — ضمن mcq / theory / analysis_exercise (≥4، medium)
سؤال مقارنة بخيارات (أ، ب، ج، د) + تعليل لماذا الباقي خاطئ

أو تمرين تحليل: «في الحالة X استخدم Y لأن Z»

## قواعد الكتل داخل الشرح

**📊 المخطط (diagrams):** 3 أقسام بالترتيب: ما هذا المخطط؟ + جدول العُقد + جدول الروابط + بلوك `diagram`. الأنواع المصرّح بها لهذه المادة: `circuit`, `waveform`, `block-diagram`, `characteristic-curve` — لا تستخدم أنواعاً خارج هذه القائمة. انظر SCHEMA.md §Diagram.
> ⚠️ **تنبيه توافق (يتطلب مراجعة يدوية):** هذه الأنواع الأربعة غير موجودة ضمن قائمة `type:` العامة الموثّقة في SCHEMA.md v1.0 (`flowchart | bpmn | decision-tree | dfd | usecase | class | activity`) — وهي قائمة موجّهة لمخططات هندسة البرمجيات ولا تصلح لدارات إلكترونية. تأكد من ملف SCHEMA.md v1.0 الفعلي قبل الاعتماد على هذا القسم: إما أنه يدعم فعلياً أنواع الدارات هذه (وحينها القائمة أعلاه صحيحة)، أو يجب تحديثه ليدعمها، أو يجب تعديل subject-brief لاستخدام أقرب قيمة مسموحة. لا تُدرج أي مخطط بنوع غير مؤكد التوافق مع البارسر.

**الفهم الخاطئ ❌ / الفهم الصحيح ✅ (compare):** سطر واحد لكل منهما.

**Callouts:** `#### مهم للامتحان ⚠️:` / `#### نقطة مهمة ⚠️:` / `#### ملاحظة:` / `#### الدرس المستفاد:` — blockquote يتبع كل منها.

**🤔 تفعيل الفهم (think_prompt):** استخدمه ≥3 مرات في كل محاضرة.

**fill_gaps:** موضع ناقص — ضع `_______` أو `(N)` مكان القيمة/الخطوة الناقصة.

**qa_inline:** `**Q{N}:**` سؤال / `A:` إجابة.

**💡 التشبيه (analogy):** بعد كل مفهوم مجرّد: جملة واحدة من الحياة اليومية + «وجه الشبه: X = Y». استخدمه بكثرة. انظر SCHEMA.md §Analogy.

**⚖️ المقايضة (trade_off):** جدول: المزايا | العيوب | متى تختاره. للحالات التي لا يوجد فيها خطأ وصواب — لا تستخدم `compare` block هنا. انظر SCHEMA.md §Trade-off.

**🔄 قبل / بعد (before_after):** حالة الدارة/القيمة قبل التغيير + بعده + جملة «ماذا تغيّر؟». انظر SCHEMA.md §Before-After.

**🔍 تتبع التنفيذ (trace، inline):** المدخل + جدول الخطوات (أعمدته تختلف حسب الموضوع — للدارات مثل: الخطوة | القانون المستخدم | القيمة) + النتيجة. انظر SCHEMA.md §Trace.

**📐 المعادلة (equations):** LaTeX داخل `$$` (نمط عرض block). يتبعها **الشرح:** بمعنى كل رمز. الرموز الشائعة لهذه المادة: `V` (الجهد)، `I` (التيار)، `R` (المقاومة)، `P` (القدرة)، `E, W` (الطاقة)، `C` (السعة)، `L` (المحاثة)، `Z` (الممانعة)، `f` (التردد). لا تخلط أكثر من معادلة غير مرتبطة في نفس البلوك. انظر SCHEMA.md §Equation.
